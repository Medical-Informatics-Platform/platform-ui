# Repository Context

This directory contains durable context for humans and AI coding agents working on the MIP Platform UI.

## Files
- `architecture.md`: system design, UI/data/auth flow, and module boundaries.
- `module-index.md`: directory and module ownership.
- `commands.md`: install, run, build, test, Docker, and CI command reference.
- `testing.md`: testing strategy and validation matrix.
- `conventions.md`: confirmed coding patterns and repo norms.
- `risk-register.md`: fragile, security-sensitive, or high-review areas.
- `decision-log.md`: lightweight architectural decision log.
- `code-review-checklist.md`: checklist for human and AI review.
- `ai-task-templates.md`: reusable prompts for future agents.

Read this context before non-trivial edits, then inspect the source files and tests for the exact area being changed.
